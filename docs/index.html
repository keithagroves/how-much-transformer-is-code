Interpretability · field report

How Much of a Transformer Is Code?

Which computations in a small language model can be replaced by explicit surrogates, at what cost, under a stated rule vocabulary — and where that stops. Short answer: a third of attention heads become positional templates and a fifth of MLP layers become context-free tables, but at a substantial cost (perplexity roughly doubles), and the fraction is a dial, not a constant.

Abstract We ask how much of a small transformer's computation can be replaced by explicit surrogates, and at what cost. On Qwen3-0.6B and Pythia-410M we fit two families — attention heads as compact positional templates (a mixture of a few named columns) and MLP layers as context-free token tables (each token's mean output) — install them, and heal only the 65k normalization gains. We keep the two apart: the templates are genuinely compact programs; the token table is memoization (millions of learned numbers), so it measures context-independence, not program structure. At a chosen tolerance, 36% of heads are template-replaceable and 21% of MLP layers context-free-replaceable — counts of architectural units, not of parameters or FLOPs, so we do not sum them into one number. The cost is substantial: against a fair baseline (an identically-healed no-code model) it is +0.88 nats on held-out text — perplexity roughly doubles (≈29 → ≈55); the raw-intact figure is +0.64 [+0.57, +0.71]. This is lossy surgery at a stated tolerance, not a free decomposition, and healing adapts a hybrid around the surrogate — substitutability, not proof the original network implemented the same rule. Controls make the surgery meaningful: positional templates tie the fitted code (the substitutable majority is plumbing, not content), while zeroing or position-scrambling before the same heal is far worse (+2.2 vs +0.7 nats), so the surrogate carries real position-specific function. The fraction is not a constant — it rises with the input's verbatim structure (fiction < WikiText < rigid schemas), and a post-heal-ranked selection beats the cheapest-first floor. The residue — which entity a head selects — resists every rule vocabulary we tried (recency, coreference, syntax, co-occurrence, a learned rank-1 direction) once the attention sink is controlled; a query edit shows a low-dim direction is causally used but does not name the choice, so we call it un-named, not smooth — it could be higher-dimensional, nonlinear, or a circuit spanning components. The honest result is a dependency, not a number: what admits a short executable description depends on the surrogate family, the data distribution, the selection procedure, the adaptation budget, and the tolerated loss. Our sharpest first answers died under their own controls (a cross-family "constant"; a first-entity rule that was the attention sink in costume).

Setup

A trained transformer is a pile of tuned weights; the question is how much of what those weights do is expressible as ordinary code. We test this by substitution. For a target component — an attention head or an MLP — we fit a readable surrogate, replace the component with it, and measure the damage to next-token loss on held-out text. Attention surrogates are least-squares mixtures of nameable columns: attention to the start token, to fixed offsets, to the last punctuation, to earlier occurrences of the current token, and to the follower of the longest previous match (an induction rule). MLP surrogates are lookup tables mapping each token to its mean output vector. After substitution we heal: fine-tune only the RMSNorm gains (65,536 of 596M parameters) with the code organs active, which recovers distribution shift without touching any weight matrix. Everything is evaluated against two references — the intact model and a zero-ablation of the same components.

By "code" we mean an operation you can name and write down — not necessarily a short one, and the two surrogate families sit at opposite ends of that scale. The attention surrogate is genuinely compact: a mixture over a fixed vocabulary of 25 named columns (start token, sixteen fixed offsets, last sentence boundary, duplicate-token and induction followers), a median of 24 nonzero coefficients per head — legible and small, a few dozen numbers you could read aloud. The MLP surrogate is the opposite: a token-indexed table of mean output vectors, ~3,400 entries of 1024 dimensions (~3.5M numbers) per substituted layer. That is legible as an operation — "look up this token's average effect and emit it" — but it is not compressible, and a skeptic is right that such a table can carry a great deal of learned data without being manually specifiable. So "code" collapses three separable axes, and it is worth pulling them apart: explicitness (can the operation be named?), compressibility (how few parameters describe it?), and substitutability (how much loss when it replaces the learned component?). The attention templates score on all three — named, tiny, and swappable. The MLP tables score on explicitness and substitutability but not compressibility: they establish that the layer is context-free (memoization of a per-token map), not that it is program-like. Because these come apart, we avoid the single word "code" for the combined claim and instead report two named quantities — template-replaceable attention and context-free MLPs — measured, and caveated, separately.

How much substitutes, and the two controls

Replacing the cheapest 160 of 448 attention heads plus 6 of 28 MLP layers — 36% and 21%, selected cheapest-first at a tolerated loss, so read these as one operating point and a floor set by the heuristic, not an intrinsic fraction of the model (a later section shows a smarter joint search substitutes more for the same loss) — costs +1.61 nats on held-out natural text before healing; retraining only the 65k RMSNorm gains brings that to +0.67, so the heal repairs about 58% of the damage. That +1.61 decomposes as +0.77 from the heads, +0.48 from the MLPs, and +0.36 of head–MLP interaction on top. Healing changes which part dominates: healed separately, heads-only falls to +0.14 and MLPs-only to +0.28 (the heal repairs 82% of the head damage), yet the full hybrid still heals to only +0.70 — so the residual cost is almost entirely the interaction (+0.29), the coupling between coded heads and coded MLPs that the per-channel norm heal cannot fully reconcile, not the organs in isolation. Norm gains can rescale and reroute residual-stream contributions, so the fair question is whether the heal manufactures the function or merely reseats the code's output — which is what the controls decide. (The healed figure holds out-of-sample: pooling eight chunks that trained neither the lookup tables nor the norm heal — and were not used to early-stop it — the healed damage is +0.64 nats, 95% CI [+0.57, +0.71], essentially the single-chunk +0.67. The unhealed decomposition above is from the headline chunk; what generalizes is the healed number.) A direct control settles whether the heal manufactures function: re-run it against the same code with its outputs permuted across positions — right statistics, wrong per-position function. Under an identical fresh-heal protocol, real code heals to +0.70 nats while the position-shuffled code heals only to +2.21 (95% CI [+2.08, +2.33]), approximately returning to the zero-ablation level (+2.12). Because the norm gains are per-channel they cannot undo a position permutation, so the recovery is functional, not the heal papering over scrambled code. (Substitution order does not matter: healing gradually, carrying the norms forward, ties healing all at once — +0.702 vs +0.705.)

Read the headline against the base rate: intact held-out loss is 3.38 nats, so +0.64 is a real degradation — perplexity rises from ≈29 to ≈55, close to a doubling — not near-parity. The claim is about what is substitutable at all at a fixed tolerance, and the controls show the recovery is functional; the number is not a free lunch. We report the +0.6–0.7 nat band as the operating point because that is where the norm-heal has converged and the zero, positional, and shuffle controls separate cleanly; tighten the tolerance and the substitutable set shrinks, loosen it and the headline inflates.

There is one more asymmetry to net out, and it moves the number. The healed hybrid fine-tuned 65k norm gains on the corpus; the intact reference did not. Running the same heal on the intact model with no code installed improves its held-out loss by −0.24 nats (95% CI [−0.27, −0.21]) — pure domain adaptation, a discount the healed-code number collects for free against the raw-intact reference. Measured instead against an identically-healed no-code baseline — the fair comparison — the code substitution costs about +0.88 nats, not +0.64. We report both, and quote +0.64 as the raw-intact figure with this offset stated. The controls are unaffected: the zero-ablation and shuffled-code conditions are healed on the same corpus and share the same discount, so their gaps from the code condition — the load-bearing comparisons — stand exactly as reported. "The cheapest 160 heads" invites a circularity worry — of course the easiest heads to replace are the template-like ones. The full cost curve both concedes and complicates it. Individually, the model's heads are massively redundant: 427 of 448 cost under 0.02 nats to replace on their own, and 121 are free or negative (removing them alone does nothing, or helps). The whole +1.61 damage is therefore not a sum of solo costs — the cheapest 160 sum to −0.39 nats solo — but an interaction effect that emerges only when many are swapped at once. So the selection criterion (solo cost) and the thing healing must repair (joint damage) are different quantities, and the genuinely expensive per-head function lives in a thin tail: the two costliest heads alone exceed 0.1 nats each, and they are induction heads, not positional ones.

Sweeping the budget makes the "one operating point, not a constant" claim concrete. Substituting the cheapest-first k heads and measuring the damage traces a sharply convex curve: the first ~40 heads are essentially free, and the cost only accelerates as the budget reaches the expensive tail. So a smaller fraction genuinely costs less — steeply so — and 36% is a chosen tolerance on a rising curve.

If solo cost and joint damage are different quantities, is cheapest-solo-first even the right way to pick the 160? We checked at the boundary: a head's individual solo cost correlates only +0.14 with what it actually costs in context given the other 159, and several un-coded heads have negative marginal cost — adding them lowers loss. So the heuristic is demonstrably not optimal. Building a same-budget set that swaps the worst-contributing coded heads for the best un-coded ones cuts the unhealed damage from +0.68 to +0.54 (~21%; heads only, MLPs held fixed to isolate the head-selection question — hence +0.68 here rather than the +1.61 heads-plus-MLPs figure above), and the curve is still falling where we stopped. The improvement is real but modest — far below the 0.45 nats a naive additive account predicts, because the same interactions that motivate the question also eat most of the gain. The honest reading is that our "36% of heads" undercounts what a smarter joint search could substitute for the same loss; the reported figure is a floor set by a simple heuristic, not a frontier. Measuring that gap directly: heal a 200-head superset, rank every head by its post-heal marginal harm (leave-one-out on the healed model — the criterion that matters, since healing rescues most head damage), then re-heal the best subset. The 160 least-harmful heads heal to +0.57 against the fixed cheapest-160's +0.70 — a 19% better frontier at the same budget — the best 120 reach +0.45, and the 47 heads that are individually non-harmful heal to just +0.33. The floor is real, and a post-heal-ranked selection clearly beats it; the substitutable frontier is wider than the cheapest-first heuristic finds. And the positional-tie result is not an artifact of that heuristic: re-running the content-free control on the joint-improved set leaves it a dead tie (full code +0.63 vs positional-only +0.63, healed), so the better-chosen heads are also mostly positional — the joint search pulls in redundant positional machinery, not the content-bearing induction circuits, which stay in the expensive tail it never reaches.

This deserves stating without hedging, and a budget sweep settles it. Dropping the two content columns (duplicate-token and match-rule) from the fitted surrogate and re-measuring across the whole range, the fitted and positional-only curves never separate: the gap is +0.003, +0.002, +0.005, +0.003, +0.007 nats at 40, 80, 120, 160, 200 heads — indistinguishable from zero everywhere. So it is not that content ties at the chosen 36%; content buys essentially nothing anywhere in the substitutable range. The honest headline for the attention half is that 36% of heads are replaceable by purely positional templates; the fitted content terms earn their keep only in the ceiling experiments on repeated text, where induction heads do real copying — and those heads sit almost entirely in the expensive, un-substituted tail. So the "96% of verbatim copying" figure below is a ceiling measurement (a match-rule column reconstructing induction on repeated sequences), not a component of the +0.64 substitution; we keep the two separate rather than let the copy-algorithm's success borrow credit for the positional headline.

The MLP result is anatomically clean and replicates across both model families: the middle of the network is nearly-free token lookup (one Qwen layer has negative substitution cost), while composition concentrates in the first two and last two layers. "MLPs as key-value memories" is not just true on average — it is a map, and the map has a shape. It is worth naming what the token-keyed table actually measures: context-independence — how much of a layer's output is a function of the current token alone, ignoring the rest of the stream. Read that way, "21% of MLP layers substitute" says a fifth of the layers are largely context-free per-token enrichment, which is a cleaner and more falsifiable claim than "run as code." Its compressibility is a separate axis we flag but do not fully resolve here: the table holds ~3,400 token entries, and whether a low-rank or top-k-frequent truncation preserves the result is the measurement that would decide how program-like it really is.

The ceiling, and why it is not a constant

Isolating induction heads and masking their true attention to only rule-nameable positions recovers a fixed share of their function. On a single fiction chunk that share came out ~32% for Qwen and ~29% for Pythia — a tidy cross-family agreement we briefly believed. A robustness suite (three models, two domains, eight chunks each) dissolved it: the fraction is domain-dependent, and the Qwen number in particular was an artifact of one chunk. Measured across eight, Qwen's fiction recovery falls to −2% (95% CI [−12%, +8%]) — no recoverable fraction is detectable under this rule vocabulary, with an upper confidence bound around 8%: on that model's hardest case the nameable share is at most marginal. Pythia's fiction holds up near ~46%. So there was never a cross-family constant; there was a noisy point estimate that happened to agree. Holding the model fixed and varying only the evaluation text isolates the dependence on the input distribution — it rules out model identity as the explanation, though selected-domain comparison is not itself a controlled intervention:

On our repeated-sequence benchmark, nearly all measured induction function flows through rule-nameable positions: in this setting the algorithmic copying behavior is almost fully code-expressible. Away from it, the fraction is a matter of degree, set by the input.

What the residue is — and how much is a limit of our rule language

The attention our first vocabulary could not name flows, on natural text, to a diffuse cloud over the story's entities — which character or object is currently in focus. Two questions follow: is that cloud nameable by a richer rule, and is it low-dimensional?

On the first, we widened the vocabulary with an entity-anchored rule (a cheap parser tags named entities) and re-measured the ceiling, against a control that adds the same number of random columns. Entity positions carry genuinely more of the head's function than random ones — Qwen recovers 40% of the induction gap from entity columns versus 19% from random, Pythia 67% versus 59%. The entity-minus-random gap is a paired +21% (95% CI [+10%, +43%]) on Qwen and +8% ([+3%, +14%]) on Pythia — both clear of zero over a chunk bootstrap, though Pythia's margin is thin — so a real part of what a narrow positional/match vocabulary misses is nameable as "attend to the entity candidates." The lesson is that the ceiling is partly a ceiling on the rule language: widen the vocabulary and it moves. But it does not vanish, and the strict rule "attend to the most recent entity" adds almost nothing (+4–5%). What a rule can name is where the candidates are; which one the head selects is left over.

The second question — whether the selection is low-dimensional — had a tidy first answer that turns out to be another sink artifact, so we state it and its collapse together. Fitting a rank-r readout of the residual stream, recency alone captured 31% of the diffuse attention mass and a single learned direction seemed to nearly double it to 61%, suggesting a clean decomposition: a nameable "attend to the candidates" part plus a low-rank "which candidate" direction. But that number came from the positions where the head's attention is most concentrated — the sink-adjacent ones — and, as the sink control below shows, it does not survive.

Reassembling part of a head's attention through its value matrix scores worse than deleting the head — attention does not decompose into reassemblable parts — so we intervene on the query instead. We reconstruct a head's true attention from the residual stream and its own weights (exact to one part in a million), edit the query at a single position along the learned selection direction, and recompute the real softmax over unchanged keys. Attention shifts toward the entities the direction predicts (correlation 0.41 with the readout's key-side score, against 0.73 for the readout's own prediction); an equal-norm random direction does nothing (−0.08); the effect scales linearly with edit size. The direction appears to be used by the head, not merely decodable from it: a cluster bootstrap over the six heads puts the true-minus-random gap at +0.49, 95% interval [+0.35, +0.64], clear of zero. It replicates cross-family: porting the same reconstruction to Pythia's GPT-NeoX circuit (fused QKV, partial rotary, no per-head QK-norm; reconstruction again exact to ~1e-6) gives a stronger effect on its six induction heads — correlation 0.63 against −0.19 for a random direction, gap +0.82, 95% interval [+0.46, +1.16]. A cluster bootstrap on six clusters is anti-conservative, so we lean on an exact test instead: on both models all six heads individually show a positive true-minus-random gap (6/6), which a two-sided sign test and an exact sign-flip permutation test both put at p = 0.031 — significant without trusting the interval width. So the causal claim is no longer one model. Still, this is the paper's thinnest leg — six heads per model, and a 0.41–0.63 correlation against the readouts' own 0.73–0.82 leaves most of the steering unexplained — so we read it as a real but partial effect, not a full account.

A tempting rule then seemed to close the story. Asking which readable rule best predicts which entity a head selects — its top-attended candidate — one wins overwhelmingly over recency and frequency across both families: attend to the first entity introduced in the passage, matched 83% of the time on Qwen and 92% on Pythia. Widening the candidate window only sharpened it, ruling out a fixed positional lag. It looked as though even the "which one" selection had a name.

It was too neat, and a control breaks it. The first entity in a passage sits near the start token, and these induction heads park half to four-fifths of their mass on the start as an off-state — the attention sink. So we forced the question: zero out the sink, renormalize over the real context, and require the first entity to appear genuinely mid-passage. The effect vanishes on both models — across 281 forced mid-passage cases (Qwen, six heads), the head's renormalized argmax lands on the first entity in exactly 0 of them (0/281, not a rounded small fraction): once it is not the sink, it is never selected.

What survives is weaker, and we already had it: when the head attends into context at all it lands on an entity most of the time — the candidate-set claim holds — but which entity it picks has no simple name. With the sink removed and the attention renormalized over the real context, we scored a whole vocabulary of readable rules against the head's actual choice — and went past hand-heuristics to a serious linguistic toolkit: recency, salience, "most recent re-mention," literal coreference, the current clause's grammatical subject (a spaCy dependency parse), and full coreference resolved by a neural model (fastcoref) that links each pronoun and mention to its entity. None of them clears a weak recency bias. The best predictor is recency itself at roughly 0.09–0.14 top-1 — two-to-five times chance, but nowhere near a rule — and the two linguistically serious candidates land below it (clause-subject 0.04–0.05, neural coreference 0.06–0.07) on both models, at >90% coverage, so it is not that the rules rarely apply. The retracted primacy rule, meanwhile, falls to chance exactly as the sink account predicts. And this is not merely a top-1 effect: scored distributionally — how well a soft model reproduces the head's whole attention distribution over candidates, not just its argmax — the story is the same, on both models. A soft weighting of nameable features (recency, salience, clause subject, coreference) reproduces the distribution no better than recency alone. Nor does the query-conditional rule a skeptic reaches for next — "attend to the entity that co-occurred with the current query token," active on a third of the cases — add anything over recency (mass-overlap 0.333 vs 0.331). Even the learned rank-1 direction, re-scored under the sink control, fails to beat recency: regularize it and it climbs monotonically toward recency-only and never above, so the encouraging 61% from earlier was — like primacy — an artifact of the concentrated, sink-adjacent positions. So within every rule family we could name — hand-heuristics, a neural-coreference-and-syntax toolkit, a soft weighting of all of them, a query-conditional co-occurrence rule, and a learned low-rank residual direction — none, scored by argmax or distributionally, beats a weak recency bias. That is a bounded negative, not a proof of absence: no rule in this vocabulary, under this soft-model class, at this scale reconstructs the selection. A query edit along the learned direction does causally nudge attention, so the direction is used — but it neither dominates nor reconstructs the selection. The residue is smooth as far as we could measure, after a serious attempt to name it, not merely un-named so far. This is where "rules all the way down" stops, and the lesson the project keeps relearning is stamped on it: in small models an apparent symbolic rule must clear a positional and attention-sink baseline before it is believed — the one rule that looked like it named the residue was the sink.

Does the shape survive scale?

Everything above lives at ≤0.6B, and the natural worry is that the structure is a small-model artifact. As a first probe we ran the cheap headline checks on Qwen3-4B — the same family, roughly seven times larger. The shape holds, and nothing inverts. Induction is still there and still concentrated (a 13.7-nat repeated-sequence collapse; about 5% of heads carry it, the same fraction as 0.6B). The MLP substitutability map is intact — expensive first and last layers, a cheap deep middle — and if anything the redundancy grows: a quarter of the 4B MLP layers are free or beneficial to replace with a token lookup, against a handful at 0.6B. Per-head importance stays thin-tailed: 88% of the 4B heads are individually near-free to ablate, the function concentrated in a small tail. And the domain-dependent ceiling reappears — induction barely matters on fiction (its gap is near zero, as at 0.6B), while on WikiText and structured text 58% of it is rule-nameable.

Two things shift with scale, both worth noting. The ceiling's ordering compresses: WikiText catches up to structured text (both 58%) rather than trailing it as at 0.6B, and induction leans even more lightly on fiction — consistent with a bigger model handling narrative through more distributed, less copy-like machinery. This is a first look, not the full suite, and the caveat matters for what it does not cover: these are the cheap diagnostics (induction concentration, the MLP map, the ablation tail, the domain ceiling), run in bf16 with a cruder lookup surrogate on a single model. The thesis-critical measurement — the healed substitution number, how much loss the code-plus-heal actually costs — was not run at 4B. So what replicates at scale is the shape, not the headline: the map, the U-shape, the thin tail, and the data-dependent ceiling are not peculiarities of the smallest models, but whether a healed hybrid holds up at 4B is untested.

What is new, and what is replication

Induction heads, copy-suppression, attention sinks, and MLPs-as-memories are established; that much of attention is replaceable by simple patterns echoes prior head-simplification work, which our positional-only control confirms from inside our own pipeline. To our knowledge the new contributions are these: the substitution-ceiling method and its finding that the nameable fraction is a property of the text's structure rather than a constant of the model, and that the ceiling is partly a property of the rule language rather than the network; the cross-family MLP substitutability map, whose shape replicates at 4B; the double-controlled replacement result, where a zero control establishes that the code carries function and a positional control localizes which templates carry content — verified on both the cheapest-solo and the joint-search set; the observation that the reported fraction is a floor set by the selection heuristic, not a frontier (solo cost correlates only +0.14 with in-context marginal cost, and a better-chosen same-budget set substitutes for less loss); and, most pointedly, the hardened residue claim — the "which entity" selection survives every nameable rule we tried, symbolic or soft, once the attention sink is controlled for, with its most seductive symbolic explanation (first-entity primacy) unmasked as the sink in costume. Taken together these are less a percentage than a method: a way to measure which computations admit a short executable description, at what cost, under a stated vocabulary — with the replaceable fraction falling out of those choices rather than the model alone.

Limitations and next steps

  • Mostly small models (≤0.6B), short context, single-digit chunk counts; ablation gaps on natural text are small (~0.1 nats), so downstream ratios are noisy. The 4B check above shows the qualitative shape survives one 7× step, but the careful measurements — healing, the full ceiling, the query-edit — were not repeated there, and nothing here reaches the sizes people deploy. Larger models with long context would both firm up the replication and make the functional questions well-posed.
  • Healing gives the network room to reroute; the shuffle and zero controls bound what that room buys (it recovers function, not distribution) and the −0.24-nat domain-adaptation offset is why we report the fair +0.88. The caveat those controls leave open: healing produces a hybrid adapted around the surrogate, so this measures substitutability, not that the original network implemented the rule.
  • The clean causal test — train twin models on regular vs. natural data and compare their nameability — failed at our scale: from-scratch 17M-parameter models never crossed the induction phase change. That experiment is compute-gated, not disproven.
  • The causal query-edit is six heads per model, now on two families (Qwen gap +0.49, 95% CI [+0.35, +0.64]; Pythia +0.82, [+0.46, +1.16]). The cluster bootstrap at n=6 is anti-conservative, so significance rests instead on an exact test — 6/6 heads positive on both models, sign and sign-flip permutation p = 0.031. The intervals are wide and the steering is partial — it shows a low-dimensional direction is used, not a tight effect size or the full shape of the attention it steers.
  • The which-entity residue resists every rule we tried, but that is a bounded negative (argued above): it holds in this vocabulary, one soft-model class, at this scale, and rests on ~six induction heads per model. It could be higher-dimensional, nonlinear, or a circuit spanning components rather than featureless — "un-named," not proven smooth. Given the many exploratory rule families, a confirmatory untouched holdout would strengthen it; and the remaining attack is scale, not more rules.
  • Is code better than same-budget pruning? We deleted a set chosen for prunability (the 160 lowest-ablation-cost heads, 53 different from the code set) and healed: it costs +3.81 nats — worse than deleting the code set (+2.12), because solo cost mispredicts joint damage (the +0.14 correlation). Code + heal (+0.70) beats both by 3–5×, so the code carries function no same-budget deletion recovers. Caveat: we did not run a full joint-optimized pruning search, which could beat +2.12; the strongest pruning baseline we measured is still the code-set zero-ablation.
  • Is the MLP "code" compressible? Partly. The full ~3,400-entry table costs +0.51 nats (MLP-only, unhealed); a rank-128 SVD or a top-1000-token truncation ≈ matches it (+0.55, +0.57), a ~10× reduction, but small truncations (rank-8, top-100) lose ~0.2 nats. So the table is moderately low-rank — not a tiny program, but not the full 3.5M numbers either.
  • The most immediately useful direction needs no new science: an audit tool that reports, for any model, what fraction of its attention and MLPs is nameable algorithm — the analysis is the deliverable.
